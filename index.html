Git 是什么？

Git 是一款常用的版本管理软件，在企业生产中用得极其广泛。对于普通用户，用得最多的当然就是 GitHub 或者 Gitee 仓库的管理啦！

安装

前往 Git 官方下载页面，选择您的操作系统，下载安装即可。

配置

安装完成后，打开 Git Bash，输入以下命令设置 GitHub 账户和邮箱：


git config --global user.name "你的GitHub用户名"
git config --global user.email "你的GitHub注册邮箱"

输入下列命令查看是否设置成功：


git config user.name
git config user.email

本地仓库与远程仓库

  • 本地仓库：存放在电脑本地的文件夹；
  • 远程仓库：存放在 GitHub/Gitee 上的仓库。

利用 Git，可以顺利地将本地仓库推送到远程仓库，或从远程仓库拉取更新，非常便捷。

配置 SSH 秘钥


ssh-keygen -t rsa -C "你的GitHub注册邮箱"

一路回车即可，完成后在 C:\Users\你的用户名\.ssh 找到 id_rsa.pub 文件，将内容复制到 GitHub 的 SSH and GPG keys 页面，添加为新的 SSH key。

检测是否成功：


ssh git@github.com

若出现 successfully，说明绑定完成。

仓库克隆

在目标文件夹中打开 Git Bash，输入：


git clone git@github.com:用户名/仓库名.git

即可将远程仓库完整下载到本地。

仓库推送

  1. 在 GitHub 创建一个新仓库（可选择是否添加 README）。
  2. 在本地初始化并提交代码：

git init
git add .
git commit -m "首次提交"
git push

仓库拉取

当远程仓库被修改后，本地需要同步更新：


git pull

这样本地与远程就保持一致，避免推送冲突。

总结

通过本教程，你已经掌握了 Git 的最基本操作：安装、配置、SSH 秘钥、仓库克隆、推送与拉取。

未来你可能会需要更多分支管理和协作的技巧，那时可以继续学习进阶的 Git 知识。

为什么需要分支？

在团队开发中，每个人可能会同时开发不同的功能。如果大家直接在主分支 (main) 上提交，容易造成冲突和混乱。

分支 就像是复制一份主线代码，你可以在分支上自由修改，完成后再合并回主线。

常用分支命令


# 查看所有分支
git branch

# 创建新分支
git branch feature-login

# 切换到新分支
git checkout feature-login

# 创建并切换
git checkout -b feature-login

# 删除本地分支
git branch -d feature-login

推送与远程分支


# 推送本地分支到远程
git push -u origin feature-login

# 拉取远程分支
git checkout -b feature-login origin/feature-login

分支合并

完成开发后，将功能分支合并到主分支：


# 切回主分支
git checkout main

# 合并功能分支
git merge feature-login

合并后建议删除功能分支，保持仓库整洁。

解决冲突

当两个分支修改了同一文件的同一位置时，合并时会产生冲突，Git 会标记冲突区域：


<<<<<<< HEAD
这是主分支的内容
=======
这是功能分支的内容
>>>>>>> feature-login

手动修改为需要的内容后，再提交一次：


git add 文件名
git commit -m "解决合并冲突"

多人协作推荐流程

  1. 从主分支 main 拉取最新代码。
  2. 基于最新代码新建一个功能分支。
  3. 在功能分支上开发并提交。
  4. 推送功能分支到远程，并发起 Pull Request (PR)。
  5. 代码审查通过后，合并到主分支。

Rebase 与 Merge

Merge：直接把分支合并，保留分支历史，日志可能较复杂。

Rebase：把你的提交“平移”到主分支的最新提交之后，保持历史更整洁。


# 将功能分支基于 main 分支最新提交
git checkout feature-login
git rebase main

总结

通过学习分支和协作，你可以：

  • 独立开发新功能而不影响主分支
  • 用分支隔离不同任务
  • 通过合并或 rebase 保持代码历史清晰
  • 利用 PR 流程提升团队协作效率

这是成为 Git 高级用户 的必经之路 🚀

GitHub 注册与创建存储库教学

一、注册 GitHub 账号

  • 准备一个常用邮箱（Gmail、Outlook、QQ 邮箱均可）。
  • 打开官网：https://github.com ，点击右上角 “Sign up”。
  • 填写邮箱、密码、用户名，完成拼图验证。
  • 去邮箱收取验证码，输入到 GitHub 完成验证。
  • 选择套餐：免费版 Free 即可。
  • 设置使用偏好，可以跳过。
  • 注册完成后进入 GitHub 首页。

    • 二、注册后推荐设置

  • 编辑个人资料：点击头像 → Your profile → Edit profile。
  • 设置 SSH Key，方便免密上传代码。
  • 开启两步验证（2FA）提升安全性。

    • 三、常见问题

  • 收不到验证邮件 → 检查垃圾邮箱或换个邮箱。
  • 用户名重复 → 换一个或在后面加数字。
  • 密码太简单 → 必须包含大小写字母、数字和符号。

    • 四、创建存储库（Repository）

  • 登录 GitHub。
  • 点击右上角 “+” → 选择 “New repository”。
  • 填写仓库信息：
  • - Repository name：例如 my-first-repo。
  • - Description：填写简介（选填）。
  • - Visibility：Public（公开）或 Private（私有）。
  • - 初始化选项：可勾选 Add a README file。
  • 点击 “Create repository” 创建。

    • 五、仓库创建完成后

  • 仓库页面地址如：https://github.com/用户名/my-first-repo。
  • 可以上传文件（Upload files）。
  • 可以克隆到本地（Clone）。
  • 可以在线新建文件（Add file → Create new file）。
